Chp 1. Getting Started
#lingo ##terms #lingo ##application #lingo ##features
On machines other than Windows, Mac and Linux, you may have to interface with LINGO through the means of a command-line prompt.
Objective Function is a formula that expresses exactly what it is you want to optimize.
Variables are the quantities you have under your control.
variables are sometimes also called decision variables.
The goal of optimization is to find the values of a model's variables that generate the best value for the objective function, subject to any limiting conditions placed on the variables.
constraints are the limiting conditions expressed as functions of the models variables.
expressions must be terminated with a semicolon.
Comments begin with an exclamation point (!) and end with a semicolon (;)
LINGO does not distinguish between uppercase and lowercase in variable names.
variable names in LINGO, must begin with an alphabetic character (A- Z). Subsequent characters may be either alphabetic, numeric (0-9), or the underscore (_).
Names may be up to 64 characters in length.
If your model does not have any nonlinear constraints, then any locally optimal solution will also be a global optimum.
If your model has one or more nonlinear constraints, then any locally optimal solution may not be the best solution to your model.
it is always preferred to formulate a model using only linear constraints whenever possible.
The Infeasibility field lists the amount that all the constraints in the model are violated by. It does not track the amount of any violations on variable bounds.
It is possible for the Infeasibility field to be zero while the current solution is infeasible due to violated variable bounds.
The .LG4 file format is a binary RTF file format.
Reduced cost: the amount that the objective coefficient of the variable would have to improve before it would become profitable to give the variable in question a positive value in the optimal solution.

Chp 6. Command Line Commands
#lingo ##application #lingo ##keywords
there is a command window accessible from the lingo GUI. It is used for writing and execute command scripts.
`TAKE`: runs a command script from an external file
`DIVERT`: opens a file for receiving output
`RVRT`: closes a file previously opened with DlVERT
`SAVE`: saves the current model to disk
`DEBUG`: tracks down formulation errors in infeasible and unbounded models
`GO`: Runs the current model
`ALTER` : edits the contents of the model
`DELETE`: deletes a selected row of the model
`TERSE`: output level
`!` for comment
`QUIT`: closes LINGO
Chp. 8. Interfacing with external files
#lingo ##application
`@FILE` imports data from external text files anywhere in the model. syntax:`@FILE(‘filename’)`
LINGO takes text from this file until it encounters either the end-of-file mark or a LINGO end-of-record mark `~`.
subsequent `@FILE` references in the same model with the same file name, make LINGO to resume taking text at the point where it left off.
LINGO Command Scripts
#lingo ##terms #lingo ##application
A LINGO command script is any text file containing a series of LINGO commands. You can think of these commands as a macro language that allows you to automate the running of frequently used commands and/or models.
To run a command script in Windows versions of LINGO, use the File|Take Commands command
`SET TERSEO`: control the amount of output LINGO generates. There are four settings available:

  1. `TERSEO 1`: Verbose—Causes LINGO to display the maximum amount of output, including full solution reports.
  2. `TERSEO 2`: Terse—Less output than Verbose, with full solution reports suppressed. This is a good output level if you tend to solve large models. LINGO also suppresses Export Summary Reports generated when exporting data to spreadsheets or databases.
  3. `TERSEO 3`: Errors Only—All output is suppressed, with the exception of error messages
  4. `TERSEO 4`: Nothing—LINGO suppresses all output. This level may be useful when taking advantage of the programming capabilities in LINGO, in which case, you will add statements to your model to generate all required output.
  5. The default setting for TERSEO is 1, or verbose mode.
`ECHOIN` When you run a LINGO command script with the TAKE command, the commands LINGO processes are normally not displayed. If you would like the commands echoed to your screen, set the ECHOIN parameter to 1. This can be a useful feature when you are trying to develop and debug a LINGO command script
`RunLingo` : Windows and Linux versions of LINGO include a utility program called RunLingo. RunLingo can be used to process LINGO script files, however, RunLingo does not include the front-end graphical interface found in the interactive version of LINGO. RunLingo is invoked from the command line, where you may enter a script file name.
command scripts are generally stored as `LTF` files.
